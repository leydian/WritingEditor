# 오류 메시지 표준안 (P0-3)

기준 일자: 2026-02-16

## 1. 원칙

1. 사용자 행동 지시가 포함되어야 한다.
2. 원인 추정 문구는 짧게 유지한다.
3. 내부 구현명(RPC 이름, 스택 등)은 직접 노출하지 않는다.

## 2. 공통 포맷

`[작업명] 실패: [핵심 원인]. [다음 행동]`

예시:

- `동기화 실패: 네트워크 연결을 확인할 수 없습니다. 잠시 후 다시 시도하세요.`

## 3. 시나리오별 기본 문구

1. 탈퇴 실패
- `회원 탈퇴 실패: 계정 확인에 실패했습니다. 이메일과 비밀번호를 다시 확인하세요.`
- `회원 탈퇴 실패: 서버 요청이 완료되지 않았습니다. 잠시 후 다시 시도하세요.`

2. 익명 전환 실패
- `계정 전환 실패: 입력한 이메일 형식을 확인하세요.`
- `계정 전환 실패: 인증 정책으로 즉시 로그인이 제한되었습니다. 이메일 인증 후 다시 로그인하세요.`

3. 동기화 실패
- `동기화 실패: 네트워크 연결을 확인할 수 없습니다. 연결 후 다시 시도하세요.`
- `동기화 실패: 서버 저장에 실패했습니다. 잠시 후 다시 시도하세요.`

## 4. 구현 메모

1. 동일 오류는 동일 문구로 매핑한다.
2. 콘솔 로그는 상세하게 남기되, UI 메시지는 간결하게 유지한다.
3. 다국어 확장 가능성을 위해 메시지 상수 테이블로 분리하는 것을 권장한다.
