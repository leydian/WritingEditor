# PART 02. 에디터 생산성 기능

작성일: 2026-02-16

## 1) 목표

- 빠른 실행(명령 팔레트), 안전한 되돌리기(히스토리 안전복원) 중심으로 작성 생산성 향상.

## 2) 주요 변경

### 명령 팔레트
- `Ctrl/Cmd + K` 열기
- 명령 검색/선택/실행, 최근 실행 명령 우선 정렬
- 모바일 닫기 UX 보강(닫기 버튼 + 백드롭 클릭 닫기)
- 관련 커밋: `7cc12f4`, `9a76573`

### 히스토리 강화
- 변경량 미리보기(글자/문단 delta)
- “현재 상태 백업 후 복원” 안전복원 플로우
- 관련 커밋: `7cc12f4`

### 아웃라인 기능 이력
- 초기 도입 후 요구사항에 따라 제거
- 관련 커밋: 도입 `7cc12f4`, 제거 `5613115`

## 3) 상태/테스트 보강

- 상태 스키마 확장: `stateVersion`, `ui.commandPalette`, history meta
- 테스트 업데이트: `tests/state-utils.test.js`

## 4) 영향 파일

- `app.js` (팔레트/히스토리 UI 및 이벤트)
- `state-utils.js` (상태 정규화/히스토리 메타)
- `tests/state-utils.test.js`
- `index.html`, `styles.css`

## 5) 현재 동작 기준

- 팔레트는 단축키/버튼으로 열림
- 모바일에서도 닫기 버튼 및 바깥 클릭 닫기 가능
- 히스토리는 복원 전에 자동 백업 엔트리 생성
