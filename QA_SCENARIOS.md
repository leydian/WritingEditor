# QA 시나리오 (P0 안정화)

기준 일자: 2026-02-16  
목적: 인증/탈퇴/익명전환과 핵심 회귀 포인트를 고정한다.

## 1. 인증 E2E

### A-1 이메일 회원가입

1. 로그인 화면에서 이메일/비밀번호 입력
2. `회원가입` 클릭
3. 성공 메시지/로그인 상태 확인

기대 결과:

- 정상 계정 생성
- 로그인 후 에디터 진입

### A-2 이메일 로그인

1. 로그아웃 상태에서 기존 계정으로 로그인
2. 앱 상태 로드 확인

기대 결과:

- 로그인 성공
- 사용자 데이터 정상 로드

### A-3 익명 시작

1. 로그인 화면에서 `익명으로 시작` 클릭
2. 상단에 익명 사용자용 `회원가입` 버튼 노출 확인

기대 결과:

- 익명 세션 생성
- 익명 사용자 UI 정책 적용

### A-4 익명 -> 이메일 전환

1. 익명 로그인 상태에서 `회원가입` 클릭
2. 이메일/비밀번호 입력 후 전환 실행

기대 결과:

- 전환 성공 시 일반 계정 상태로 변경
- 정책상 인증 필요 시 안내 메시지 정상 노출

## 2. 탈퇴 E2E

### W-1 일반 계정 탈퇴

1. 로그인 후 로그아웃
2. 로그인 화면에서 `회원 탈퇴` 진입
3. 재로그인 정보 입력 + 체크박스 + `회원탈퇴` 문구 입력 후 실행

기대 결과:

- 계정 삭제 성공
- 원격 상태(`editor_states`) 삭제
- 로컬 상태 키 정리

### W-2 익명 로그아웃 자동탈퇴

1. 익명 로그인 상태에서 로그아웃 클릭

기대 결과:

- 자동 탈퇴 흐름 실행
- 원격/로컬 데이터 정리
- 로그인 화면 복귀

## 3. 회귀 시나리오

### R-1 히스토리

1. 문서 생성/수정/삭제 후 히스토리 항목 생성 확인
2. 히스토리 검색 사용
3. 히스토리 복원 실행

기대 결과:

- 히스토리 최대 10개 유지
- 복원 후 에디터 상태 정상 복구
- 히스토리 목록 자체는 유지

### R-2 동기화

1. 로컬 편집 후 수동 동기화(`지금 동기화`)
2. 자동 동기화 대기 또는 로그 확인

기대 결과:

- 원격 저장 성공
- 오류 시 사용자에게 명확한 실패 메시지 노출

### R-3 모바일 미니모드

1. 화면 폭 `<=900px`에서 접속
2. `문서목록` 토글, `달력` 토글 각각 점검
3. 바깥 영역 터치로 드로어 닫힘 확인

기대 결과:

- 문서목록/달력 동시 오픈 방지
- 분할 버튼 모바일에서 숨김
